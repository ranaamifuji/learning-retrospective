# 初心者FAQ拡張 BRIEF

## 目的

バイブコーディング中に実際につまずき、エージェントへ確認して解決した内容を、同じ場所で迷った初心者が自力で次の一手を選べる形で残す。

## 画面

- 「初心者FAQ」カテゴリー
- 質問の一覧
- 各質問の回答、理由、確認手順、成功の目印
- 専門用語の補足
- 関連する一言プロンプト
- 必要な項目だけ画面キャプチャと説明文

## 操作

1. 「初心者FAQ」を選ぶとFAQだけが表示される
2. 質問を選ぶと回答の詳細が開閉する
3. キーワードを入力すると、質問・回答・用語を横断検索できる
4. 画面キャプチャがある場合は、説明文と一緒に内容を確認できる

## FAQの記録形式

1. **問** — 初心者が実際に迷った言葉
2. **回答** — 最初に結論を短く示す
3. **理由** — なぜそうなるかを説明する
4. **確認手順** — 次に何をすればよいかを番号付きで示す
5. **成功の目印** — 解決したと判断できる表示や状態
6. **専門用語** — その項目で必要な言葉だけ補足する
7. **関連プロンプト** — エージェントへ確認するときの再利用可能な一言
8. **画面キャプチャ** — 文章だけでは場所や状態を判別しにくい場合のみ付ける

## 画面キャプチャのルール

- 保存先は `assets/faq/`
- アカウント名、実メール、パスワード、認証コード、リカバリーコード、APIキー、個人ファイルのパス、通知を写さない
- 秘密情報が写った画像は加工して使わず、撮り直す
- 画像には内容が分かる代替テキストと説明文を付ける
- 文章と成功の目印だけで十分な項目には画像を付けない

## 完了条件

- [x] FAQカテゴリーで絞り込める
- [x] 各項目に「問→回答→理由」がある
- [x] 各項目に確認手順と成功の目印がある
- [x] 必要な専門用語がその場で分かる
- [x] FAQの内容がキーワード検索の対象になる
- [x] 今後の学習終了時にも同じ形式で追記できる
- [x] 秘密情報・個人情報を含まない

## やらないこと

- 実際に起きていないつまずきを作らない
- 日次ログ全文を公開サイトへ転載しない
- 画面キャプチャを必須にしない
- 特に指示がなければGitHubへpushしない（公開依頼時のみ）
